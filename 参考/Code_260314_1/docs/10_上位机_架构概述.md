## SerialManager — 数据总线核心

**源文件**: `src/APP/core/serial_manager.py` (848 行)

### 职责

```
SerialManager 是上位机的数据中枢:

  发送方向:
    Tab → send_command(cmd, payload) → CDC 串口 → FPGA
    
  接收方向:
    CH340 串口 → SerialRxThread → _on_data_received()
      → 状态机分流 → 各专用 Signal → 各 Tab 槽函数
```

### 信号定义

| 信号 | 参数类型 | 触发场景 |
|------|---------|---------|
| `connected` | `(str, str)` | 双串口连接成功 |
| `disconnected` | `()` | 连接断开 |
| `data_received` | `(bytes)` | 所有 CH340 原始数据 |
| `command_sent` | `(int, bytes, str)` | 每条 CDC 命令发送后 |
| `log_message` | `(str)` | 日志信息 (调试Tab) |
| `adc_data_received` | `(list)` | 以太网 UDP ADC 数据 |
| `frequency_data_received` | `(bytes)` | 8 字节 CH1+CH2 频率 |
| `adc_capture_completed` | `()` | Buffer 模式采集完成 |
| `dsa_measurement_received` | `(int, dict)` | DSA 20 字节测量结果 |
| `can_data_received` | `(bytes)` | CAN 接收帧 |
| `bode_data_received` | `(int, float, float, float, float, float)` | Bode 扫频数据 |

### CH340 数据分流状态机

```
SerialRxThread.run() → data_received(bytes)
  → SerialManager._on_data_received(data)
    │
    ├── DSA 状态机 (dsa_response_state=WAIT_DATA)
    │   └── 累积 20 字节 → dsa_measurement_received Signal
    │
    ├── 频率状态机 (freq_response_state=WAIT_DATA)
    │   └── 累积 8 字节 → frequency_data_received Signal
    │
    ├── CAN 检测 (首字节 0x00/0x01)
    │   └── 累积 CAN 帧 → can_data_received Signal
    │
    ├── Bode 数据包 (21/49 字节)
    │   └── 累积 bode 包 → bode_data_received Signal
    │
    └── 应答帧 / 未知数据
        └── log_message (调试Tab显示)
```

> **源码**: `src/APP/core/serial_manager.py:260-550` (完整分流逻辑)

### 双串口设计

```python
# src/APP/core/serial_manager.py:104-148
def connect(self, tx_port, rx_port, baud_rate=115200):
    self.serial_tx = serial.Serial(tx_port, baud_rate)  # CDC 命令
    self.serial_rx = serial.Serial(rx_port, baud_rate)  # CH340 接收
    self.rx_thread = SerialRxThread(self.serial_rx)     # 独立接收线程
    self.rx_thread.data_received.connect(self._on_data_received)
    self.rx_thread.start()
```

- `serial_tx`: 仅发送 CDC 命令帧
- `serial_rx`: 仅接收 CH340 数据，通过独立线程避免阻塞 UI
- `SerialRxThread`: QThread 子类，每 10ms 轮询一次串口缓冲区

### send_command 流程

```python
# src/APP/core/serial_manager.py:193-258
def send_command(self, cmd, payload=b""):
    frame = generate_command(cmd, payload)  # 组装 CDC 帧
    self.serial_tx.write(frame)             # 发送
    self.command_sent.emit(cmd, payload, cmd_name)  # 信号通知
    self.log_message.emit(f"[TX→CDC] {cmd_name}")   # 日志
```

> `generate_command()` 定义在 `src/APP/core/serial_protocol.py`，按照 `55 AA | cmd | len | payload | cs` 格式组装。

## 开发规范

| 规则 | 说明 |
|------|------|
| 文件命名 | `snake_case.py` |
| 类命名 | `PascalCase` |
| 导入顺序 | 标准库 → 第三方 (PySide6/numpy/pyqtgraph) → 本地模块 (core/tabs/utils) |
| Tab 解耦 | Tab 之间不互相引用，通过 SerialManager 信号通信 |
| 槽函数 | 命名 `on_<signal_name>` |
| 线程 | 串口和 UDP 使用独立线程，UI 更新通过 Signal→Slot 自动跨线程 |
