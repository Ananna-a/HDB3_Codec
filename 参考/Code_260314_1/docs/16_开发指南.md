# 开发指南

## FPGA 端开发环境

### 必需工具

| 工具 | 版本 | 说明 |
|------|------|------|
| Gowin EDA (云源软件) | 最新版 | FPGA 综合 / 布局布线 / 比特流生成 |
| Programmer | Gowin 自带 | 下载比特流到 FPGA |

### 开发流程

```
1. 打开工程: 双击 acm2108_ddr3_CDC.gprj
2. 修改 Verilog 代码
3. 综合 (Synthesize) — 使用 GowinSyn 引擎
4. 布局布线 (Place & Route)
5. 生成比特流 (Generate Bitstream)
6. 下载到 FPGA (Programmer)
```

### 工程配置要点

- 综合引擎: **GowinSyn**（非 Synplify Pro）
- 顶层模块: `debugger_top`
- 引脚约束: `src/acm2108_ddr3_CDC.cst` (441 行)
- 注册源文件: 78 个 `.v` 文件在 `.gprj` 中注册
- **新增文件需在 `.gprj` 中手动注册路径** — 否则综合时被忽略

### 新增 CDC 命令

在 `debugger_top.v` 中需要两处修改：

1. **白名单注册** (`cmd_valid_flag` 组合逻辑, `src/debugger_top.v:1276-1389`):
```verilog
// 在对应区间添加新命令码
8'hXX: cmd_valid_flag = 1'b1;
```

2. **命令处理** (main always 块, `cmd_done && cmd_valid_flag` case):
```verilog
8'hXX: begin
    // 解析 payload, 设置寄存器
end
```

如果命令需要 Payload 参数，建议使用独立锁存器模式 (参考 0x26/0x60/0xB0 实现)。

### IP 核修改

所有 `src/ip/` 下文件由 Gowin IP Generator 生成:
- PLL: 修改输出时钟频率需重新生成
- FIFO: 修改深度/位宽需重新生成
- DDR3: 修改时序参数需重新生成
- **不可直接编辑 IP `.v` 文件**，下次生成会覆盖

## 上位机端开发环境

### 环境搭建

```powershell
# 安装 Python 3.8+
# 安装依赖
pip install PySide6 numpy pyqtgraph pyserial

# 可选
pip install scipy        # AWG 编辑器三次样条插值

# 运行
cd src\APP
python main_app.py
```

### 运行要求

- **必须在 `src/APP/` 目录下运行** — 模块使用相对导入
- 连接到 FPGA: CDC COM (FX2) + CH340 COM (接收)
- 以太网: 可选的单独 UDP 连接 (示波器)
- 串口波特率: 115200 bps

### Cython UDP 加速

如果 UDP 接收性能不足:

```powershell
cd src\APP\core
python setup_fast_udp.py build_ext --inplace
```

需要 Visual Studio Build Tools (Windows C 编译器)。

### 开发规范

```
文件命名: snake_case.py
类命名:   PascalCase
导入顺序: 标准库 → 第三方 → 本地模块
Tab 解耦: 通过 Signal/Slot 间接通信, 不直接引用
槽函数:   on_<signal_name>
信号命名: 动词_名词 (data_received, command_sent)
```

## 调试技巧

### FPGA 端

| 调试手段 | 说明 |
|---------|------|
| 8 个 LED | 心跳 + DDR3 初始化 + 各模块状态 |
| 数码管 | 频率测量值 / 温度值 / 自定义显示 (0x03) |
| CH340 应答 | 每条命令返回 `AA 55 01 XX status 00 CS` |
| cmd_error 脉冲 | 校验和错误指示 (LED 未直接连接, 应答 status=0x01) |
| 帧头错误 | cdc_cmd_parser 自动重新搜索 `55 AA` |

### 上位机端

| 调试手段 | 说明 |
|---------|------|
| 调试日志 Tab | 显示所有应答帧和系统消息 |
| `log_message` Signal | SerialManager 发出的所有日志 |
| 串口监视 | 使用第三方软件监视 CDC/CH340 数据 |
| UDP 统计 | EthernetReceiver 提供丢包率和包计数 |

### 常见调试流程

```
1. 确认 PLL 锁定 → LED[0] 闪烁 (心跳)
2. 发送 0x03 命令 → 数码管显示变化 → CDC 通道 OK
3. 查看调试日志 → 有 AA 55 应答帧 → CH340 通道 OK
4. DDR3 LED[1] 亮 → DDR3 初始化完成
5. 示波器模式: LED[4] 亮 → ADC 采集中
```
