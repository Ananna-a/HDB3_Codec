# HDB3 编解码器

这是一个面向教学与实机演示的 HDB3（High Density Bipolar 3）编解码项目，包含 FPGA 下位机、C# WPF 上位机、DAC 波形显示和串口通信。

- 上下位机串口通信、编码/解码应答、DAC 波形显示均已实机验证
- 当前仅剩 ModelSim 仿真文件与波形验证待补充

## 项目能做什么

| 功能 | 说明 |
|------|------|
| 编码 | 二进制序列转换为 HDB3 符号序列 |
| 解码 | HDB3 符号序列转换为二进制序列 |
| DAC 输出 | DA0 显示编码波形，DA1 显示译码波形 |
| 回环自检 | 编码结果自动送入译码器，检查两路结果是否一致 |
| 串口应答 | FPGA 通过同一串口返回结果帧 |
| LED 调试 | 8 个 LED 显示心跳、收发、命令状态、忙状态和播放状态 |
| ModelSim 仿真 | 功能已实机通过，仿真作为最后补充验证项 |

## 硬件平台

- FPGA 芯片：GW5AT-138B（GW5AT-LV138PG484AC1/I0）
- 开发板：ACX720
- 系统时钟：50 MHz
- 串口：115200 bps，8N1
- DAC：双通道 8 位，板上码值映射为 `0x00` 正电压、`0x80` 零电平、`0xFF` 负电压

## 运行方法

### 上位机

直接双击发布好的程序：

[HDB3_App.exe](d:/Desktop/HDB3_Codec/src/APP/bin/Release/net10.0-windows/win-x64/publish/HDB3_App.exe)

```powershell
cd src/APP
dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true
```

### FPGA

1. 用 Gowin EDA 打开 `HDB3_Codec.gprj`
2. 顶层模块选择 `hdb3_top`
3. 运行综合、布局布线、生成比特流
4. 下载到 ACX720 开发板
5. 打开上位机连接串口，发送编码/解码命令即可观察返回结果和 DAC 波形

## 现在采用的 HDB3 规则

本项目采用的是**奇偶法**实现。

- 每次遇到连续 4 个 `0`，就根据“自上次 `V` 以来普通 `1` 的个数奇偶”决定输出 `B00V` 还是 `000V`
- 替换后会同步更新极性状态，保证后续普通 `1` 继续按 AMI 规则交替

这和课堂里常见的“第一次四连零先直接放 `V`，后面如果连续同号 `V` 再用 `B` 矫正”的讲法，目标是一样的，都是为了保持极性交替和直流平衡。区别只是对第一个四连零的描述方式不同。本仓库里的代码、文档和上位机期望结果都以当前奇偶法为准。

## DAC 波形说明

- DA0：HDB3 编码波形，双极性归零 RZ，前半个码元出脉冲，后半个码元归零
- DA1：译码 bit 波形，整码元宽度保持不变
- 两路通道共用同一个 `DA_Clk`，所以符号边界是对齐的
- 示波器上 DA0 和 DA1 形状不同是预期现象，不是错位
