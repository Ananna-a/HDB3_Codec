## 1. WPF 是什么

**WPF**（Windows Presentation Foundation）是微软的桌面 UI 框架。你可以把它理解成"用 XML 画界面 + C# 写逻辑"。

一个 WPF 项目由两类文件构成：

| 文件类型 | 作用 | 类比 |
|----------|------|------|
| `.xaml` | 画界面（布局、颜色、按钮） | HTML |
| `.cs` | 写逻辑（点击事件、数据处理） | JavaScript |

**关键区别**：WPF 不是 HTML/CSS，它有自己的一套规则，但思想上类似——XAML 负责"长什么样"，C# 负责"干什么事"。

### WPF vs 你熟悉的 HTML

| HTML 概念 | WPF 对应 |
|-----------|----------|
| `<div>` | `<Grid>`, `<StackPanel>`, `<Border>` |
| CSS class | `Style` + `StaticResource` |
| JavaScript onclick | `Command="{Binding XxxCmd}"` |
| `document.getElementById()` | `{Binding PropertyName}` |
| React state | `INotifyPropertyChanged` |

## 3. 启动流程：从点击到窗口出现

1. **`App.xaml`** 指定启动窗口：`StartupUri="MainWindow.xaml"`
2. **`App.xaml.cs`** 加载全局资源（主题色、按钮样式、转换器）
3. **`MainWindow.xaml`** 创建窗口，在顶部实例化 ViewModel：
   ```xml
   <Window.DataContext>
       <vm:MainViewModel/>
   </Window.DataContext>
   ```
   这一行是关键——它告诉 WPF："这个窗口的所有数据绑定，都去 MainViewModel 里找"。
4. **`MainViewModel` 构造函数**运行：
   - 创建 `SerialService`（串口服务）
   - 创建三个 `RelayCommand`（连接、编码、解码按钮的命令）
   - 刷新可用串口列表
5. 窗口显示，用户开始交互。

## 4. 核心机制一：XAML 界面

XAML 是 XML 方言，每个标签对应一个 C# 类。

### 4.1 布局容器

本项目用到三种容器：

```xml
<!-- Grid: 表格布局，分行/列 -->
<Grid>
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto"/>   <!-- 高度随内容 -->
        <RowDefinition Height="*"/>      <!-- 占满剩余空间 -->

<!-- StackPanel: 垂直/水平堆叠 -->

    <Button Content="A"/>
    <Button Content="B"/>

<!-- Border: 带边框的容器（可设圆角、背景色） -->
<Border Background="#161B22" CornerRadius="10" BorderThickness="1">
    ...
</Border>
```

### 4.2 控件

```xml
<!-- 文本 -->
<TextBlock Text="编码" FontSize="16" Foreground="#58A6FF"/>

<!-- 输入框 -->
<TextBox Text="{Binding EncInput}" FontFamily="Consolas"/>

<!-- 按钮 -->

<!-- 下拉框 -->

<!-- 动态列表（结果展示用的那个 WrapPanel 符号流） -->

    <ItemsControl.ItemTemplate>

```

### 4.3 Style（样式）

定义在 `App.xaml` 的 `<Application.Resources>` 中，然后在 XAML 里引用：

```xml
<!-- App.xaml 中定义 -->

    <Setter Property="Height" Value="34"/>
    <Setter Property="Foreground" Value="White"/>
</Style>

<!-- MainWindow.xaml 中使用 -->
<Button Style="{StaticResource BtnEncode}" Content="发送编码"/>
```

相当于 CSS 的 class。`x:Key` 是名字，`{StaticResource xxx}` 是引用方式。

## 5. 核心机制二：数据绑定

数据绑定是 WPF 最重要的概念。它让界面自动跟随数据变化。

### 5.1 基本语法

```xml
<!-- 单向：界面显示 C# 变量的值 -->
<TextBlock Text="{Binding EncStatus}"/>
<!-- 当 MainViewModel.EncStatus 变化时，TextBlock 自动更新 -->

<!-- 双向：输入框修改 → C# 变量同步变化 -->
<TextBox Text="{Binding EncInput, UpdateSourceTrigger=PropertyChanged}"/>
<!-- PropertyChanged = 每次按键都同步，不是等焦点离开 -->
```

### 5.2 绑定源头：DataContext

```xml
<Window.DataContext>
    <vm:MainViewModel/>  <!-- 这一行设置了绑定的"根" -->
</Window.DataContext>
```

所有 `{Binding Xxx}` 都从这个 MainViewModel 实例上找属性。

### 5.3 通知机制：INotifyPropertyChanged

C# 属性变化后，必须"通知"界面刷新。关键代码：

```csharp
public class MainViewModel : INotifyPropertyChanged
{
    private string _encInput = "";
    public string EncInput
    {
        get => _encInput;
        set
        {
            _encInput = value;
            OnPropertyChanged();  // 喊一声："我变了！"
        }
    }

    public event PropertyChangedEventHandler PropertyChanged;
    private void OnPropertyChanged([CallerMemberName] string n = null)
        => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(n));
}
```

**规则**：只要界面需要显示或响应的属性，setter 里必须调 `OnPropertyChanged()`。

### 5.4 ObservableCollection

普通 `List<T>` 添加/删除元素时不会通知界面。要用 `ObservableCollection<T>`：

```csharp
public ObservableCollection<ObservableSymbol> EncActual { get; } = new();
// .Clear() 或 .Add() 时界面自动刷新
```

## 6. 核心机制三：MVVM 模式

MVVM = Model-View-ViewModel，三层分离：

| 层 | 是什么 | 本项目文件 | 规则 |
|----|--------|-----------|------|
| **Model** | 数据和算法 | `HDB3Codec.cs` | 不引用 ViewModel，不知道界面的存在 |
| **View** | 界面 | `MainWindow.xaml` | 不知道 Model，只绑定 ViewModel |
| **ViewModel** | 中间人 | `MainViewModel.cs` | 持有 Model，提供数据给 View |

```
用户点击按钮
    → View (XAML Button Command绑定)
    → ViewModel (MainViewModel.SendEncode())
    → Model (HDB3Codec.Encode())
    → Service (SerialService.SendCommand())
    → 硬件 (UART → FPGA)

FPGA返回数据
    → Service (SerialService.OnDataReceived → FrameReceived事件)
    → ViewModel (OnFrame回调 → 更新属性)
    → View (数据绑定自动刷新界面)
```

**最重要的纪律**：ViewModel 绝对不要引用 View 的控件（如 `TextBox`、`Button`），只能操作数据和命令。

## 7. 核心机制四：命令与按钮

WPF 按钮不直接写 `Click="Button_Click"`（虽然也可以），而是用 `Command` 绑定：

### 7.1 使用方式

```xml
<Button Content="发送编码" Command="{Binding EncodeCmd}"/>
```

```csharp
// ViewModel 中
public ICommand EncodeCmd { get; }

public MainViewModel()
{
    EncodeCmd = new RelayCommand(_ => SendEncode(), _ => !IsBusy && _serial.IsOpen);
    //                               ↑ 点击时执行         ↑ 是否可点击（CanExecute）
}
```

`RelayCommand` 封装了两个函数：
- `_ => SendEncode()` — 点击时调用
- `_ => !IsBusy && _serial.IsOpen` — 返回 false 时按钮自动灰掉（IsEnabled=false）

### 7.2 RelayCommand 实现

```csharp
public class RelayCommand : ICommand
{
    private readonly Action<object> _exec;
    private readonly Func<object, bool> _canExec;

    public RelayCommand(Action<object> exec, Func<object, bool> canExec = null)
    { _exec = exec; _canExec = canExec; }

    public bool CanExecute(object p) => _canExec?.Invoke(p) ?? true;
    public void Execute(object p) => _exec(p);
    public event EventHandler CanExecuteChanged { add { } remove { } }
}
```

### 8.5 MainViewModel.cs（视图模型 — 核心）

```csharp
public class MainViewModel : INotifyPropertyChanged
{
    // ===== 依赖服务 =====
    private readonly SerialService _serial;

    // ===== 构造函数 =====
    public MainViewModel()
    {
        _serial = new SerialService(disp);
        _serial.FrameReceived += OnFrame;     // 订阅FPGA应答事件

        EncodeCmd = new RelayCommand(_ => SendEncode(),
                                     _ => !IsBusy && _serial.IsOpen);
        RefreshPorts();
    }
}
```

**编码流程**（`SendEncode()` 方法）：

```
1. 检查忙状态 → IsBusy==true 则 return
2. 从 EncInput 字符串提取 0/1 字符
3. 长度校验（>0, ≤2048）
4. char[] → bool[] (chars.Select(c => c == '1'))
5. HDB3Codec.Encode(bits) → 软件算出期望符号
6. HDB3Codec.PackBits(bits) → 打包成字节
7. 组装 payload: [bit_cnt_l][bit_cnt_h][packed_bytes...]
8. _serial.SendCommand(0x01, payload) → UART发送
9. IsBusy = true → 按钮灰掉
10. 显示期望结果到 EncExpected 列表
```

**FPGA应答处理**（`OnFrame()` 方法）：

```
SerialService.DataReceived (后台线程)
    → 解析 AA 55 帧
    → FrameReceived 事件 (切回UI线程)
    → OnFrame(cmd, status, payload)
        ├─ status != 0 → 错误日志
        ├─ cmd == 0x01 → HandleEncResponse()
        │    └─ 逐符号比对 payload vs EncExpected → 着色
        └─ cmd == 0x02 → HandleDecResponse()
             └─ 逐bit比对
```

### 8.6 HDB3Codec.cs（数据模型）

纯算法，不依赖任何 UI 或服务：

```csharp
public static class HDB3Codec
{
    public static byte[] Encode(bool[] bits);     // 编码
    public static bool[] Decode(byte[] symbols);   // 解码
    public static byte[] PackBits(bool[] bits);    // bool[] → byte[]
    public static bool[] UnpackBits(byte[] packed, int bitCnt);
    public static string SymbolToString(byte sym); // 0x01 → "+1"
    public static byte StringToSymbol(string s);   // "+1" → 0x01
}
```

### 8.7 SerialService.cs（串口服务）

```csharp
public class SerialService : IDisposable
{
    // ===== 公开接口 =====
    public bool IsOpen { get; }
    public string[] PortNames { get; }
    public event Action<byte, byte, byte[]> FrameReceived;  // cmd, status, payload
    public void Open(string portName, int baudRate = 115200);
    public void Close();
    public void SendCommand(byte cmd, byte[] payload);

    // ===== 内部实现 =====
    // OnDataReceived: SerialPort 后台线程触发
    //   → 字节流存入 _rxBuf
    //   → 循环扫描 AA 55 帧头
    //   → 提取 cmd/status/len/payload
    //   → 校验和验证
    //   → 通过 SynchronizedDispatcher 切回 UI 线程
    //   → 触发 FrameReceived 事件
}
```

**线程安全问题**：`SerialPort.DataReceived` 运行在后台线程，不能直接更新 UI 控件。必须用 `Dispatcher.Invoke()` 切回 UI 线程。

```
后台线程 (SerialPort)              UI 线程 (WPF)
───────────────                    ─────────
OnDataReceived()                   Application.Current.Dispatcher
  │                                   ↑
  ├─ 解析 AA 55 帧                     │
  ├─ 校验通过                          │
  └─ _dispatcher.Invoke( ─────→  FrameReceived 事件
                                      │
                                      ├─ 更新 EncActual 列表
                                      ├─ 更新 IsBusy = false
                                      └─ UI 自动刷新
```

## 9. 数据流全景

### 编码完整流程

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 用户输入 "11001" 到 TextBox                                  │
│    → XAML: Text="{Binding EncInput, UpdateSourceTrigger=...}"   │
│    → ViewModel.EncInput = "11001"                               │
│    → OnPropertyChanged("EncInput") 通知界面                     │
├─────────────────────────────────────────────────────────────────┤
│ 2. 用户点击 [发送编码]                                          │
│    → XAML: Command="{Binding EncodeCmd}"                        │
│    → RelayCommand.Execute() → SendEncode()                      │
│    → "11001" → bool[]{true,true,false,false,true}               │
├─────────────────────────────────────────────────────────────────┤
│ 3. 软件编码（求期望值）                                         │
│    → HDB3Codec.Encode(bits)                                     │
│    → byte[]{0x01, 0x01, 0x00, 0x00, 0x01} = [+1, +1, 0, 0, +1] │
│    → EncExpected 列表显示：+1 +1 0 0 +1                          │
├─────────────────────────────────────────────────────────────────┤
│ 4. 通过串口发送到 FPGA                                          │
│    → PackBits → byte[]                                           │
│    → 组装帧: 55 AA 01 05 00 [bit_data] cs                       │
│    → SerialPort.Write(frame)                                    │
│    → IsBusy = true → 按钮灰掉                                    │
├─────────────────────────────────────────────────────────────────┤
│ 5. FPGA 处理... (编码+回环解码)                                  │
├─────────────────────────────────────────────────────────────────┤
│ 6. FPGA 应答帧到达                                              │
│    → SerialPort.DataReceived (后台线程)                          │
│    → 字节流解析 → AA 55 01 00 05 00 [+1][+1][0][0][+1] cs      │
│    → 校验通过 → Dispatcher.Invoke → FrameReceived 事件           │
├─────────────────────────────────────────────────────────────────┤
│ 7. ViewModel 处理应答                                           │
│    → OnFrame(cmd=0x01, status=0x00, payload)                    │
│    → HandleEncResponse(payload)                                  │
│    → 逐符号比对: payload[i] vs EncExpected[i]                    │
│    → EncActual 列表显示（绿色=正确，红色=错误）                   │
│    → IsBusy = false → 按钮恢复                                   │
│    → 日志: "[14:30:22] 编码应答: 5 符号, 全部正确"              │
└─────────────────────────────────────────────────────────────────┘
```

## 附录：常用 WPF 速查

### 布局

| 控件 | 效果 |
|------|------|
| `Grid` | 表格布局（最灵活） |

### 绑定模式

| Mode | 说明 |
|------|------|
| `OneWay` | ViewModel → View 单向（默认） |
| `TwoWay` | 双向同步（输入框用） |
| `OneTime` | 仅初始化时读一次 |
