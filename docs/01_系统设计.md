# HDB3 编解码器 — 系统设计文档

> 版本: v1.1 | 日期: 2026-06-17 | 状态: 代码已实现，文档已同步

---

## 1. 项目概述

HDB3（High Density Bipolar 3）编解码器，面向教学演示场景，FPGA 硬件实现 + C# WPF 上位机 + 仿真验证。

- **FPGA 芯片**: GW5AT-138B (GW5AT-LV138PG484AC1/I0)，ACX720 开发板
- **参考模板**: `参考/Code_260314_1/` — 多功能调试器项目
- **通信方式**: 单 UART（115200 bps, 8N1），命令和应答走同一 COM 口
- **用途**: 教学演示 HDB3 编码/解码原理

## 2. 功能需求

| 功能 | 说明 |
|------|------|
| 编码 | 二进制序列 → HDB3 符号序列（含 +1/-1/+V/-V/+B/-B 六种符号） |
| 解码 | HDB3 符号序列 → 二进制序列 |
| DAC 输出 | DA0 输出编码波形，DA1 输出译码波形，通过 dac_wave_ram BSRAM 循环回放 |
| 自检回环 | 编码模式下，encoder 输出自动送入 decoder，一次请求两个 DAC 都有波形 |
| 正确性对比 | 上位机 C# 软实现 HDB3 算法，FPGA 返回结果后逐符号对比，差异标红 |
| 状态指示 | led0 心跳灯 (~2Hz) |

## 4. 通信协议

### 4.1 物理层

- 接口: UART，单条 TX/RX
- 波特率: 115200 bps
- 帧格式: 8 数据位，1 停止位，无校验（8N1）
- 引脚约束: `uart_tx`=M15, `uart_rx`=J21（LVCMOS33，上拉，tx=8mA驱动）

### 4.2 帧格式

**命令帧（PC → FPGA）**
```
字节:   0      1      2      3      4      5..N+4  N+5
      +------+------+------+------+------+------+------+
      | 0x55 | 0xAA | cmd  | len_l| len_h| payload | cs |
      +------+------+------+------+------+------+------+
        帧头(2B)    命令码    载荷长度(LE16)  载荷   校验和
```

**应答帧（FPGA → PC）**
```
字节:   0      1      2      3      4      5      6..N+5  N+6
      +------+------+------+------+------+------+------+------+
      | 0xAA | 0x55 | cmd  |status| len_l| len_h| payload | cs |
      +------+------+------+------+------+------+------+------+
        帧头(2B)    命令码  状态    载荷长度(LE16)   载荷   校验和
```

**状态码 (status)**

| 值 | 含义 |
|----|------|
| 0x00 | 成功 |
| 0x01 | FPGA 忙（正在处理上一请求） |
| 0x02 | 校验和错误 |
| 0x03 | 无效命令码 |
| 0x04 | 长度/参数非法 |

**区分规则**: 帧头 `55 AA` = 命令，`AA 55` = 应答

### 4.3 校验和

命令帧: `cs = (cmd + len_l + len_h + Σpayload) & 0xFF`
应答帧: `cs = (cmd + status + len_l + len_h + Σpayload) & 0xFF`

### 4.4 命令码

| cmd | 方向 | payload | 说明 |
|-----|------|---------|------|
| 0x01 | PC→FPGA | bit_cnt_l + bit_cnt_h + packed_bits | 编码请求 |
| 0x01 | FPGA→PC | HDB3 符号序列（每字节一个符号） | 编码应答 |
| 0x02 | PC→FPGA | HDB3 符号序列（每字节一个符号） | 解码请求 |
| 0x02 | FPGA→PC | 解码比特序列（每字节一个 bit: 0x00/0x01） | 解码应答 |

### 4.5 Payload 编码

**二进制比特串 payload (编码请求)**: `[bit_cnt_l:1B] [bit_cnt_h:1B] [bit_data:N字节]`
- 前两字节：有效比特数（LE16，最大 2048）
- 后续字节：比特数据，每字节存 8 bit（MSB 在前），末尾字节不满 8 bit 时高位补 0

**HDB3 符号序列 payload**: 每符号 1 字节

| 值 | 符号 |
|----|------|
| 0x00 | 0 |
| 0x01 | +1 |
| 0x02 | -1 |
| 0x03 | +V |
| 0x04 | -V |
| 0x05 | +B |
| 0x06 | -B |

### 4.6 交互流程

```
编码: PC → 55 AA 01 xx yy [bit payload] cs
      FPGA: encoder(bits) → symbols → DA0 + decoder(symbols) → bits → DA1
      FPGA → AA 55 01 status xx yy [symbol payload] cs
      PC: HDB3Codec.Encode(bits) 软比较, 逐符号着色

解码: PC → 55 AA 02 xx yy [symbol payload] cs
      FPGA: decoder(symbols) → bits → DA1 + symbols → DA0 (显示输入符号)
      FPGA → AA 55 02 status xx yy [bit payload] cs
      PC: HDB3Codec.Decode(symbols) 软比较, 逐符号着色
```

## 5. FPGA 模块设计

### 5.1 hdb3_encoder（HDB3 编码器）

**设计模式**: 三趟批处理——顶层写入 bit_buf 后发 start 脉冲，encoder 分三个阶段完成编码：

1. **趟1 S_ENCODE**: 逐 bit 从 `bit_buf` 读取，按 AMI 规则映射为符号存入本地 `sym_buf[0:2047]`
2. **趟2 S_SUBST**: 扫描 `sym_buf`，检测连续 4 个 0 并替换为 B00V / 000V，原位修改缓冲区
3. **趟3 S_OUTPUT**: 逐符号输出 `sym_out` / `sym_valid` / `sym_wr_en`

采用本地符号缓冲区而非流式输出，彻底避免了 B00V 替换需要回写前 3 个符号的时序难题。

**接口**

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `clk` | 1 | in | 系统时钟 50MHz |
| `rst_n` | 1 | in | 异步复位（低有效） |
| `start` | 1 | in | 启动编码脉冲 |
| `total_bits` | 11 | in | 本批次有效比特数（0~2047） |
| `bit_buf_byte` | 8 | in | 从 bit_buf 读取的字节 |
| `bit_buf_addr` | 8 | out | 向顶层请求的字节地址 (bit_idx/8) |
| `sym_out` | 3 | out | 编码输出符号 |
| `sym_valid` | 1 | out | 符号有效脉冲 |
| `sym_wr_en` | 1 | out | 符号写入使能（接 buffer） |
| `done` | 1 | out | 编码完成脉冲 |

**内部寄存器**

| 寄存器 | 位宽 | 说明 |
|--------|------|------|
| `state` | 3 | 状态: IDLE→ENCODE→SUBST→OUTPUT→DONE |
| `bit_idx` | 11 | 当前处理的 bit 索引 (0 ~ total_bits-1) |
| `zero_cnt` | 2 | 连续零计数器（0~3） |
| `ami_pol` | 1 | AMI 当前极性（0=正，1=负） |
| `last_nonzero_pol` | 1 | 上一个非零脉冲的极性 |
| `pulse_parity` | 1 | 自上次 V 以来 '1' 个数奇偶性（0=偶，1=奇） |
| `subst_buf[0:3]` | 3×4 | 替换序列缓存（B00V/000V 4 符号） |
| `subst_idx` | 2 | 替换序列输出索引（0~3） |

**HDB3 编码规则 (ITU-T G.703)**
1. 输入 `1`：按 AMI 交替极性输出 `+1`/`-1`，`zero_cnt` 清零，`pulse_parity` 翻转
2. 输入 `0`（`zero_cnt < 3`）：暂存为 `0`，`zero_cnt++`
3. 输入 `0`（`zero_cnt == 3`，即连续 4 个零）：
   - `pulse_parity` 为偶 → 替换为 B00V（B 与上一个脉冲反极性，V 与 B 同极性）
   - `pulse_parity` 为奇 → 替换为 000V（V 与上一个脉冲同极性）
   - `zero_cnt` 清零，`pulse_parity` 归零

**3-bit 符号编码**

| 值 | 符号 |
|----|------|
| 3'd0 | 0 (零电平) |
| 3'd1 | +1 (正脉冲) |
| 3'd2 | -1 (负脉冲) |
| 3'd3 | +V (正破坏脉冲) |
| 3'd4 | -V (负破坏脉冲) |
| 3'd5 | +B (正平衡脉冲) |
| 3'd6 | -B (负平衡脉冲) |

**关键设计点**: 批处理模式避免了流式处理中 B00V 回写前 3 个符号的难题。顶层在 `start` 前将 bit 数据写入 bit_buf，encoder 通过 `bit_buf_addr` 请求读取。顶层用 `enc_addr` 驱动 `always @(*) enc_byte = bit_buf[enc_addr]` 提供字节数据。

### 5.2 hdb3_decoder（HDB3 解码器）

**设计模式**: 批处理——顶层持有 sym_buf，逐符号送入 decoder。

**接口**

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `clk` | 1 | in | 系统时钟 50MHz |
| `rst_n` | 1 | in | 异步复位（低有效） |
| `start` | 1 | in | 启动解码脉冲 |
| `total_syms` | 11 | in | 本批次总符号数（0~2047） |
| `sym_in` | 8 | in | 输入符号字节（低 3 bit 有效） |
| `sym_addr` | 11 | out | 当前符号地址索引 |
| `sym_avail` | 1 | in | 符号可用标志 |
| `bit_out` | 1 | out | 解码后二进制比特 |
| `bit_valid` | 1 | out | 比特有效脉冲 |
| `done` | 1 | out | 解码完成脉冲 |

**解码规则**

仅当符号为 `+1`(3'd1) 或 `-1`(3'd2) 时输出 bit=1，其他所有符号（0/+V/-V/+B/-B）均输出 bit=0。V 脉冲和 B 脉冲在 HDB3 中是由连续 4 个 0 替换产生的，它们本身不对应原始数据的 1，因此解码后必须还原为 0。

**状态**: IDLE → DECODE → DONE（3 状态）

### 5.3 dac_playback（DAC 波形播放）

**架构**: dac_playback 内部例化 2 个独立的 `dac_wave_ram` 实例（通道0=编码波形，通道1=译码波形）。

**dac_wave_ram** (`src/hdb3/dac_wave_ram.v`)
- 存储: `reg [7:0] mem[0:2047]`，GowinSyn 自动推断为 BSRAM 硬核
- 写端口: `wr_en`, `wr_addr[10:0]`, `wr_data[7:0]`
- 读端口: `rd_addr[10:0]` → `rd_data[7:0]`（读延迟 1 周期）
- 复位时 rd_data 初始化为 8'h80（0V 中点）

**dac_playback 播放控制**
- 写入阶段: 顶层通过 `wr_en0/1` + `wr_addr0/1` + `wr_data0/1` 写入符号
- `load_done` 脉冲 → 启动循环播放
- `playing` 状态: 码元时钟 tick → 地址 0→len-1→0 无限循环
- `stop` 脉冲 → 停止播放，DAC 输出归中值 8'h80
- 码元时钟分频: 50MHz / 500 = 100kHz（每个符号持续 10μs）

**电压映射**

| 符号值 | 含义 | DA 输出 | 电压 |
|--------|------|---------|------|
| 0x00 | 0 | 8'h80 | 0V |
| 0x01, 0x03, 0x05 | +1 / +V / +B | 8'hFF | +5V |
| 0x02, 0x04, 0x06 | -1 / -V / -B | 8'h00 | -5V |

通道0（编码符号）按上述映射转换；通道1（译码比特，值为 0x00/0x01）: 0x00→8'h80(0V)，其他→8'hFF(+5V)。

**接口**

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `wr_data0/1` | 8 | in | 写入数据 |
| `wr_en0/1` | 1 | in | 写使能 |
| `wr_addr0/1` | 11 | in | 写地址 |
| `load_done` | 1 | in | 启动播放脉冲 |
| `play_len0/1` | 11 | in | 播放长度（有效符号数） |
| `stop` | 1 | in | 停止播放 |
| `DA0_Data` | 8 | out | DAC 通道 A（编码波形） |
| `DA1_Data` | 8 | out | DAC 通道 B（译码波形） |
| `DA_Clk` | 1 | out | DAC 采样时钟 50MHz（双通道共用，与系统时钟同频） |

**两个 DAC 时钟**: `DA1_Clk = DA0_Clk`（顶层 assign 语句直连）

### 5.4 packet_parser（包协议解析）

**状态机**: 8 状态

```
S_IDLE → (收到 0x55) → S_HEAD2
               → (收到 0xAA) → S_CMD
               → S_LEN_L → S_LEN_H
               → (len>0 ? S_PAYLOAD : S_CS)
               → S_PAYLOAD (循环直到收完)
               → S_CS
                   ├─ cs_calc == rx_data → S_DONE (输出 cmd_done)
                   └─ 不匹配 → S_ERROR (输出 cmd_error)
S_DONE/S_ERROR → S_IDLE
```

**抗干扰**: S_HEAD2 状态收到非 0xAA 时，若为 0x55 则保持在 S_HEAD2（帧头重叠保护），否则退回 S_IDLE。

**输出方式**: payload 逐字节输出（`payload_valid` + `payload_data`），由顶层存入自己的 `payload_buf` 数组。

**接口**

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `rx_data` | 8 | in | 接收到的字节（来自 uart_byte_rx） |
| `rx_valid` | 1 | in | 字节有效脉冲 |
| `cmd_code` | 8 | out | 解析出的命令码 |
| `payload_len` | 8 | out | payload 字节数（低字节） |
| `payload_data` | 8 | out | payload 逐字节数据 |
| `payload_valid` | 1 | out | payload 字节有效脉冲（单周期） |
| `cmd_done` | 1 | out | 解析成功脉冲 |
| `cmd_error` | 1 | out | 解析失败脉冲 |

### 5.5 response_tx（应答发送）

**接口**

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `start` | 1 | in | 启动发送脉冲 |
| `resp_cmd` | 8 | in | 应答命令码 |
| `resp_status` | 8 | in | 状态码 |
| `resp_len` | 8 | in | payload 字节数 |
| `wr_data` | 8 | in | payload 写入数据 |
| `wr_addr` | 8 | in | payload 写入地址 |
| `wr_en` | 1 | in | payload 写使能 |
| `send_en` | 1 | out | UART 发送使能 |
| `send_data` | 8 | out | 待发送字节 |
| `tx_done` | 1 | in | UART 发送完成脉冲 |
| `tx_busy` | 1 | in | UART 忙标志 |
| `resp_done` | 1 | out | 应答发送完成脉冲 |

**内部**: `reg [7:0] payload_buf[0:255]` 存储应答 payload，顶层在 start 前通过 wr_en/wr_addr/wr_data 写入。

**状态机**: S_IDLE → S_SEND → S_WAIT → (byte_idx达到total) → S_DONE

**帧组装顺序**: AA → 55 → cmd → status → len_l → len_h(固定0x00) → payload[...] → cs

### 5.6 hdb3_top（顶层模块）

**端口**

| 端口 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `clk_50m` | 1 | in | 50MHz 系统时钟 |
| `rst_n` | 1 | in | 异步复位（低有效） |
| `uart_rx` | 1 | in | UART 接收引脚 |
| `uart_tx` | 1 | out | UART 发送引脚 |
| `DA0_Data` | 8 | out | DAC 通道 A 数据 |
| `DA0_Clk` | 1 | out | DAC 通道 A 时钟 |
| `DA1_Data` | 8 | out | DAC 通道 B 数据 |
| `DA1_Clk` | 1 | out | DAC 通道 B 时钟 |
| `led0` | 1 | out | LED0 心跳灯（~2Hz，低有效） |

**内部存储数组（仅顶层持有，端口全部扁平）**

| 数组 | 大小 | 用途 |
|------|------|------|
| `payload_buf` | 256×8bit | 接收 parser 输出的 payload 数据 |
| `sym_buf` | 512×8bit | 编码符号暂存（回环解码 + 应答 payload） |
| `bit_buf` | 256×8bit | 编码用比特数据（从 payload 中提取） |

**主控状态机**: 10 状态

| 状态 | 说明 |
|------|------|
| M_IDLE | 空闲，等待 cmd_done |
| M_ENC_PREP | 编码准备：从 payload 提取 bit_cnt + bit_data 到 bit_buf |
| M_ENC_RUN | 编码运行：encoder 输出符号 → DA0 + sym_buf |
| M_ENC_LOOP | 编码回环：sym_buf 逐符号送入 decoder → 解码 bit → DA1 |
| M_DEC_PREP | 解码准备：payload 符号 → DA0（显示输入） |
| M_DEC_RUN | 解码运行：decoder 输出 bit → DA1 + sym_buf |
| M_START_DAC | 启动 DAC 循环播放 |
| M_RESP_WRITE | 将 sym_buf 写入 response_tx 内部缓冲 |
| M_RESP_SEND | 触发 response_tx 发送应答帧 |
| M_ERR_SEND | 发送错误应答（无效命令码） |

**数据流（编码）**
```
payload_buf → bit_buf → encoder(bit_buf_byte/bit_buf_addr) → enc_sym
  → DA0 (dac_wr0) + sym_buf → decoder(dec_sym_in) → dec_bit
  → DA1 (dac_wr1) → DAC 播放 → sym_buf → response_tx → UART TX
```

**数据流（解码）**
```
payload_buf → DA0 (dac_wr0) + decoder(dec_sym_in) → dec_bit
  → DA1 (dac_wr1) + sym_buf → DAC 播放 → sym_buf → response_tx → UART TX
```

### 6.3 数据流

1. 用户输入二进制序列 → 点击 [发送编码]
2. MainViewModel → HDB3Codec.Encode() 算出期望 → 显示 EncExpected
3. MainViewModel → SerialService.SendCommand(0x01, payload) → UART
4. FPGA 返回 → FrameReceived(cmd=0x01, status, payload) → 解析符号 → 显示 EncActual
5. 逐符号比对 EncExpected vs EncActual，正确=绿色，错误=红色
6. 解码流程同理（cmd=0x02）
