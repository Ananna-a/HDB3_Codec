## 数据接收链路

```
FPGA → RGMII → 以太网 → PC
         ↓
UDP :6102 → EthernetReceiver.receive_loop()
  → _parse_packet_fast()
  → adc_data_received.emit(raw_data)
         ↓
OscilloscopeTab._on_adc_data(raw_data):
  解交织 → 电压映射 → RingBuffer → 定时器 20FPS → pyqtgraph 显示
```

### 双通道解交织

```python
# oscilloscope_tab.py 核心代码
CH2_data = raw[16::2]  # 取偶数索引: CH2[15:8]
CH1_data = raw[17::2]  # 取奇数索引: CH1[7:0]

# 电压映射: 8位ADC (0-255) → -5V ~ +5V
voltage_CH1 = [(byte / 255.0) * 10.0 - 5.0 for byte in CH1_data]
voltage_CH2 = [(byte / 255.0) * 10.0 - 5.0 for byte in CH2_data]
```

### 两级缓冲架构

```
EthernetReceiver (高速 UDP) → list 批量数据
  → oscope._on_adc_data() 主线程槽函数
  → ch1_buffer.append(voltages)  ← RingBuffer(500K 点, float32)
  → ch2_buffer.append(voltages)
  
QTimer(50ms=20FPS):
  → update_display()
    → ch1 = ch1_buffer.get_latest(N_display_points)
    → ch2 = ch2_buffer.get_latest(N_display_points)
    → curve.setData(ch1)  ← pyqtgraph PlotCurveItem
```

**RingBuffer 避免内存抖动**：固定大小 numpy 数组 + 环形写入指针，不分配新内存。

## 两种采集模式

### Stream 模式 (0x20 mode=0)

```
界面特征: "流模式" RadioButton选中
行为:
  发送 0x23 (启动采集)
  FPGA 连续捕获 → UDP 持续发送
  上位机 RingBuffer 循环覆盖最新 500K 点
  每 50ms 刷新 pyqtgraph 曲线
  触发配置可操作 (trigger_enable 控制自动/触发模式)
```

### Buffer 模式 (0x20 mode=1)

```
界面特征: "Buffer模式" RadioButton选中
行为:
  发送 0x20 mode=1 → 0x21 大小 → 0x22 触发 → 0x23 启动
  FPGA 等待触发 → 采集指定点数 → 自动停止
  UDP 分包发送 → 上位机逐包接收
  完成后: adc_capture_completed Signal → 界面显示"采集完成"
```

## 频率测量

```
用户点击/定时频率测量 (0x27):
  serial_manager.send_command(0x27, b"")
  
FPGA 回复:
  先应答 AA 55 01 27 00 00 CS
  再通过 CH340 发送 8 字节:
    [CH1_freq LE32][CH2_freq LE32]

上位机 SerialManager:
  freq_response_state: IDLE → WAIT_RESPONSE → WAIT_DATA
  收齐 8 字节 → frequency_data_received.emit(data)
  
OscilloscopeTab 槽函数:
  CH1_freq = struct.unpack('<I', data[0:4])[0]
  CH2_freq = struct.unpack('<I', data[4:8])[0]
  更新界面频率标签
```

> 频率数据也通过 CH340_connected → uart_tx_mux (最高优先级) 发送

## FFT 频谱分析

```python
# oscilloscope_tab.py FFT 实现
import numpy as np

def compute_fft(signal: np.ndarray, fs: float):
    """对时间域信号做 FFT"""
    N = len(signal)
    fft_data = np.fft.rfft(signal * window_function(signal))
    magnitude = np.abs(fft_data) / N
    freq_axis = np.fft.rfftfreq(N, d=1.0/fs)
    return freq_axis, magnitude
```

可选窗函数: Hanning / Hamming / Blackman / 矩形窗

**源文件**: `src/APP/utils/ring_buffer.py` (377 行)

### RingBuffer 核心设计

```python
class RingBuffer:
    """固定大小 numpy 环形缓冲, 线程安全"""
    
    def __init__(self, capacity=100000, dtype=np.float32):
        self._buffer = np.zeros(capacity, dtype=dtype)  # 预分配
        self._write_pos = 0     # 环形写入指针
        self._data_count = 0    # 有效数据量
        self._lock = threading.Lock()
    
    def append(self, data: List[float]) -> int:
        """追加数据, 跨边界自动分段写入"""
        with self._lock:
            # 环形写入: 可能分两段
            end_pos = self._write_pos + len(data)
            if end_pos <= capacity:  # 一次写入
                self._buffer[write_pos:end_pos] = data
            else:  # 环绕写入
                self._buffer[write_pos:] = data[:first_part]
                self._buffer[:second_part] = data[first_part:]
```

### PingPongBuffer — 乒乓双缓冲

```
双缓冲模式:
  Buffer A (写入)           Buffer B (显示)
  正在填充中...             上一轮的完整快照
  
  A 写满 → 交换 → B 写入, A 显示
```

适合高速流式数据场景，避免写入-读取锁竞争。
