# 系统架构

## 总体架构

三条独立物理通道实现 FPGA 与上位机的双向通信：

```
┌──────────────────────────────────────────────────────────────────────┐
│  上位机 (Python / PySide6)                                            │

│  main_app.py                                                          │

│    │   ├── serial_tx  ──→ CDC COM (仅发送)                            │
```

## 三条通信链路的详细数据路径

### 链路 1：CDC 命令下发（PC → FPGA）

```
上位机 send_command(cmd, payload)
  → serial_protocol.generate_command() 组装帧
  → serial.write(frame) → USB CDC

FPGA 侧:
  FX2 SlaveFIFO (端点2) → flagb 通知有数据
  FX2_CDC_Core RX状态机: IDLE→READ→WRITE
    → data_valid + fifo_data_in (48MHz域)
  fifo_in (异步FIFO, 48MHz→50MHz跨时钟域)
    → rx_fifo_out (50MHz域)
  cdc_cmd_parser: S_IDLE→S_HEAD2→S_CMD→S_LEN_L→S_LEN_H→S_PAYLOAD→S_CS→S_DONE/S_ERROR
    → cmd_code, cmd_payload(逐字节), cmd_payload_valid, cmd_done
  debugger_top cmd_valid_flag组合逻辑: case(cmd_code_latched) → 白名单命中=1
  main always块: cmd_done+cmd_valid_flag → case分发 → 设置子模块参数寄存器
```

> **源码位置**: `src/comm/FX2_CDC_Core.v:126-156` (RX状态机), `src/comm/cdc_cmd_parser.v:28-187` (帧解析), `src/debugger_top.v:1274-1389` (cmd_valid_flag)

### 链路 2：CH340 应答回传（FPGA → PC）

```
每个命令执行后:

简单命令:
  cmd_done上升沿 → cmd_finish → response_valid=1'b1 (单周期脉冲)
  uart_response_tx: S_PREPARE→S_WAIT→S_SEND (7字节应答帧)
    → uart_tx_data + uart_tx_send_en
  uart_tx_mux: 仲裁后 → uart_tx_data_mux
  uart_byte_tx: 波特率分频(50MHz/115200≈434周期/bit)
    → uart_tx (单线) → CH340 → USB → 上位机

子模块命令 (CAN/I2C/SPI/DS18B20):
  子模块完成后自己产生 cmd_done + 应答/数据流
  数据流 (如CAN接收帧): can_rx_report_valid → uart_tx_mux (CH_CAN_RX)
    → uart_byte_tx → CH340

上位机 SerialManager._on_data_received() 状态机分流:
  ① DSA帧 → dsa_measurement_received Signal
  ② 频率帧 → frequency_data_received Signal
  ③ CAN数据 → can_data_received Signal
  ④ Bode数据 → bode_data_received Signal
  ⑤ 应答帧 → log_message Signal (调试日志Tab)
```

> **源码位置**: `src/debugger_top.v:1392-1481` (应答生成), `src/comm/uart_response_tx.v:38-105` (应答帧组装), `src/comm/uart_tx_mux.v:124-274` (多路复用), `src/comm/uart_byte_tx.v:56-153` (UART物理层)

### 链路 3：UDP 示波器数据上报（FPGA → PC）

```
ADC_CH1/CH2 模拟输入
  → 8位并行ADC @ 50MSPS (adc_clk_180, PLL 180°相位)
  → adc_capture_stream (50MHz域)
    → 状态机: IDLE→WAIT_TRIGGER→CAPTURING→DONE
    → 可配置分频: div_set=1→50MSPS, div_set=2→25MSPS...
  → adc_dual_channel_interleaver: {CH2, CH1} 交织为16位
  → DDR3 写端口 (ddr3_ctrl_2port, 128位宽×burst 128)
  → DDR3 读端口 → 异步FIFO (50MHz→125MHz)
  → adc_eth_tx_controller: 分包1024B, 帧头5A AA, 序号+时间戳
  → eth_udp_tx_wrapper → eth_udp_tx_gmii (GMII)
  → gmii_to_rgmii → RGMII → 以太网PHY
  → 上位机 EthernetReceiver (端口6102)
    → _parse_packet_fast() → adc_data_received Signal
    → OscilloscopeTab: 解交织→电压映射→RingBuffer(500K)→20FPS pyqtgraph
```

> **源码位置**: `src/scope/ADC/adc_capture_stream.v` (ADC采集), `src/scope/ethernet/adc_eth_tx_controller.v` (UDP分包)

## 时钟域架构

```
外部 50MHz 晶振
    │
    ├── gowin_pll (IP)
    │   ├── clkout0: 125MHz  → DDS_Module_Dual (DDS核心)
    │   │                    → eth_udp_tx_gmii (GMII TX)
    │   │                    → gmii_to_rgmii (RGMII转换)
    │   ├── clkout1:  50MHz  → ADC芯片 (adc_clk_180, 180°相位)
    │   └── pll_lock         → sys_rst = ~reset_n | ~pll_lock
    │
    ├── ddr_pll (IP)
    │   ├── clkout2: 400MHz  → DDR3 控制器参考时钟
    │   └── pll_lock_ddr     → DDR3 IP 初始化
    │
    ├── clk: 50MHz → 系统主时钟域
    │   ├── debugger_top 主控制逻辑
    │   ├── cdc_cmd_parser (命令解析)
    │   ├── uart_response_tx, uart_tx_mux, uart_byte_tx (UART)
    │   ├── DDS_Param_Controller (DDS参数更新)
    │   ├── PWM / CAN / I2C / SPI / DS18B20 控制
    │   └── adc_capture_stream (ADC采集, 50MHz域)
    │   └── logic_analyzer_capture (逻辑分析, 50MHz域)
    │
    └── fx2_ifclk: 48MHz → FX2_CDC_Core (SlaveFIFO)
        └── fifo_in → 跨时钟域 → 50MHz 系统域
```

**关键跨时钟域处理**：

| 跨域路径 | 方案 | 源码 |
|---------|------|------|
| FX2 48MHz → 系统 50MHz | `fifo_in` 异步FIFO (Gowin IP) | `src/ip/fifo_in/` |
| 系统 50MHz → DDS 125MHz | `arb_wave_ram_simple` 双时钟RAM | `src/dds/arb_wave_ram_simple.v` |
| 系统 50MHz → DDR3 400MHz | `ddr3_ctrl_2port` 内部FIFO | `src/ip/ddr3_ctrl_2port/` |
| ADC 50MHz → 以太网 125MHz | `wr_data_fifo` + `rd_data_fifo` | `src/ip/wr_data_fifo/`, `src/ip/rd_data_fifo/` |
| ADC状态 50MHz→125MHz | 2级同步寄存器 (亚稳态防护) | `src/debugger_top.v:296-307` |
| CH340 接收 → 系统 50MHz | `uart_byte_rx` 16倍过采样 | `src/comm/uart_byte_rx.v:62-71` |

## 共享资源与冲突策略

| 资源 | 使用者 | 仲裁策略 |
|------|--------|---------|
| CH340 UART TX | 7类数据源 | `uart_tx_mux` 严格优先级仲裁（单字节粒度） |
| DDR3 256MB | 示波器 + 逻辑分析仪(预留) | `ddr3_ctrl_2port` 双端口，写/读并发 |
| RGMII PHY | 示波器(独占) | CAN UDP通道已禁用（缺仲裁器，注释确认于 `src/debugger_top.v:2340-2345`） |
| I2C 总线 | 通用I2C + OLED | 顶层命令锁存切换 `src/debugger_top.v:2090-2111` |
| DDS 通道A | 函数发生器 + Bode扫频(预留) | MUX: `bode_dds_enable ? bode参数 : 用户参数` (`src/debugger_top.v:1989-1999`) |

### 已验证无误的关键点

- **DDS**: 125MHz 时钟，32位相位累加器，相位偏移 `degree * 11930465`（精度 ±0.000002%），5 种 ROM + 脉冲 + 任意波形，共 7 种波形
- **PWM**: 基于 DDS 原理，三级流水线消除毛刺，占空比 16 位精度（0.0015% 步进）
- **ADC**: 两级采样率控制（`RESAMPLE_RATIO` 固定 + `div_set` 动态），50MSPS 基础，支持流/Buffer 双模式
- **DDR3**: 128 位宽 × burst 128，乒乓缓冲，流控阈值 64KB 起读、254MB 停写、32KB 急停
- **FX2_CDC_Core**: flagc 8 周期延迟（167ns），TX 三态状态机含 FLUSH 消除最后字节丢失
- **uart_byte_tx**: 自动波特率分频 `bps_DR = Clk_Freq/Baud_Rate`，`115200bps 8N1`
- **I2C 总线**: 通用 I2C 与 OLED 通过 `i2c_generic_cmd_req` 锁存信号切换，无仲裁冲突
- **CAN UDP**: `can_udp_tx` 模块已实例化，但输出未连接到 RGMII（缺以太网 TX 仲裁器），`can_udp_tx_done` 接 0
