# 通信协议

## 物理层

三条独立物理通道，分别承担不同职责：

| 通道 | 方向 | 物理接口 | 速率/时钟 | 用途 |
|------|------|---------|----------|------|
| **CDC** | PC → FPGA | FX2 USB CDC（端点2 RX, 端点6 TX） | USB 2.0 FS, IFCLK=48MHz | 命令下发 |
| **CH340** | FPGA → PC | CH340G USB-UART | 115200 bps, 8N1 | 应答帧 + 数据上报 |
| **UDP** | FPGA → PC | RGMII 以太网 → UDP/IP | 1000 Mbps | 示波器 ADC 波形数据 |

> **设计理念**：下行命令和上行应答使用不同 UART 芯片，彻底避免 RS-232 半双工冲突。大数据量（ADC波形）走独立 UDP 通道。

## CDC 命令帧（PC → FPGA）

### 帧结构

```
字节偏移   0     1     2    3      4       5..(N+4)   (N+5)
        +-----+-----+----+------+------+-----------+-------+
        | 0x55| 0xAA| cmd| len_l| len_h| payload...|  cs   |
        +-----+-----+----+------+------+-----------+-------+
          帧头         命令码   载荷长度     载荷       校验和
                         (小端序)
```

- **帧头**: `0x55 0xAA`，固定两字节
- **cmd**: 1 字节命令码（见下方完整命令码表）
- **len_l, len_h**: 2 字节小端序，payload 长度（0~65535，实际最大 ~1456）
- **payload**: 变长载荷数据
- **cs**: 1 字节校验和

### 校验和算法

```
cs = ~(cmd + len_l + len_h + Σpayload[N]) & 0xFF
```

即从 cmd 字节开始到 payload 最后一字节的累加和取反与 0xFF。

> **源码**: `src/comm/cdc_cmd_parser.v:100-146` — `checksum_calc` 从 S_CMD 开始累加 `cmd + len_l + len_h + Σpayload`，在 S_CS 状态与接收的 `checksum_rx` 比较。

### 解析状态机

```
S_IDLE → (收到0x55) → S_HEAD2 → (收到0xAA) → S_CMD
  → S_LEN_L → S_LEN_H → (len>0? S_PAYLOAD : S_CS)
  → S_PAYLOAD (循环直到 payload_cnt >= length) → S_CS
  → S_DONE (校验通过) 或 S_ERROR (校验失败)
```

特性：
- 帧头错误时自动重新搜索：`S_HEAD2` 中非 0xAA 字节如果是 0x55 则留在 S_HEAD2，否则退回 S_IDLE
- 支持零长度 payload（`len_l=0, len_h=0` 时直接从 S_LEN_H 跳到 S_CS）
- 每个有效命令码在 S_CMD 状态产生 `cmd_valid_pulse` 脉冲，用于上层锁存命令码

> **源码**: `src/comm/cdc_cmd_parser.v:67-186`

## 应答帧（FPGA → PC，CH340）

### 帧结构

```
字节偏移   0     1       2       3       4      5    6
        +-----+-----+-------+-------+-------+-----+-----+
        | 0xAA| 0x55| mod_id|func_id|status | data| cs  |
        +-----+-----+-------+-------+-------+-----+-----+
          帧头           模块ID  功能ID   状态   数据 校验和
```

- **帧头**: `0xAA 0x55` — 与命令帧头 `0x55 0xAA` 相反，便于上位机区分
- **mod_id**: 模块 ID（当前固定 `0x01` 系统模块，CAN 模块使用 `0x30`）
- **func_id**: 功能 ID（= 命令码）
- **status**: 状态码
  - `0x00` = 成功
  - `0x01` = 校验错误（CS 不匹配）
  - `0x02` = 无效命令（不在白名单）
- **data**: 附加数据字节（通用 I2C 读取/扫描结果回填）
- **cs**: 校验和

### 校验和算法

```
cs = (mod_id + func_id + status + data) & 0xFF
```

> **源码**: `src/comm/uart_response_tx.v:56-63` (帧组装), `src/debugger_top.v:1445-1476` (应答触发与状态码生成)

## UDP 示波器数据包（FPGA → PC，以太网）

### 包格式

```
+------+------+------+------+------+------+------+------+-----------+
| 0x5A | 0xAA | SEQ_H| SEQ_L| TS[4]| LEN_H| LEN_L| MODE | DATA[~1008B]
+------+------+------+------+------+------+------+------+-----------+
  帧头          包序号(大端)  时间戳   数据长度  模式   ADC采样数据
```

- **帧头**: `0x5A 0xAA`
- **SEQ**: 16 位大端序包序号（用于丢包检测）
- **TS[4]**: 32 位时间戳
- **LEN**: 16 位数据长度
- **MODE**: 模式标志
- **DATA**: ADC 采样数据（CH1/CH2 交织，`CH2[15:8], CH1[7:0]`）

> UDP 端口: **6102**。上位机 `EthernetReceiver` 绑定 `0.0.0.0:6102`。

> **源码**: `src/APP/core/ethernet_receiver.py:59-62` (帧头/包长常量), `src/scope/ethernet/adc_eth_tx_controller.v` (FPGA 端组包)

## UART 多路复用协议（CH340 通道上的数据流）

CH340 通道上除了标准 7 字节应答帧外，还传输以下几类数据流。上位机 `SerialManager._on_data_received()` 通过状态机自动识别：

| 数据类型 | 识别特征 | 长度/格式 | 优先级 |
|---------|---------|----------|:--:|
| **频率测量** | 4 字节连续 (CH1+CH2 各4B = 8B总) | `[FREQ_CH1_LE32]` `[FREQ_CH2_LE32]` | 1 (当前2) |
| **DSA 测量** | 0x66 命令后 20 字节连续 | `[channel] [freq LE32] [high LE32] [low LE32]` | 4 |
| **CAN 接收** | 首字节 0x00(标准帧) 或 0x01(扩展帧) | 变长: `[type][ID...][DLC][data0..7]` | 3 |
| **Bode 数据** | 0xB0-0xB3 命令后 49 字节 | I/Q 解调结果包 | 1 (调试期) |
| **SPI Flash** | 0x82/0x83/0x87 命令后 | 流式原始字节 | 4 |
| **DS18B20** | 0xA0/0xA1 命令后 | 2 字节温度值 | 5 |
| **蓝牙透传** | 低优先级轮询 | 变长透传 | 7 |
| **应答帧** | 标准 7 字节 `AA 55 ...` | 7 字节固定 | 6 |

### UART TX 多路复用优先级

```
正式优先级（源码注释）:
  频率 > CAN_RX > Bode > DSA = SPI > DS18B20 > 应答帧 > 蓝牙透传

当前实测优先级（调试期临时调整）:
  Bode > 频率 > CAN_RX > DSA > SPI > DS18B20 > 应答帧/蓝牙
```

单字节粒度仲裁：`uart_tx_mux` 在 IDLE 状态检查各通道 posedge，按优先级依次检查，高优先级的请求会抢占。

> **源码**: `src/comm/uart_tx_mux.v:123-274` (完整仲裁状态机 + 优先级列表)

## 上位机端 CH340 数据流识别逻辑

`SerialManager._on_data_received()` 状态机（简化）：

```
接收原始字节
  │
  ├── 检查 DSA 状态机 (dsa_response_state=WAIT_DATA)
  │   └── 累积 20 字节 → dsa_measurement_received Signal
  │
  ├── 检查频率状态机 (freq_response_state=WAIT_DATA)
  │   └── 累积 8 字节 (CH1+CH2) → frequency_data_received Signal
  │
  ├── 检查 CAN 首字节 (0x00 或 0x01)
  │   └── 累积 CAN 帧 → can_data_received Signal
  │
  ├── 检查 Bode 数据包
  │   └── 累积 21/49 字节 → bode_data_received Signal
  │
  └── 其余 → log_message (显示为调试日志)
```

> **源码**: `src/APP/core/serial_manager.py:260-550` (`_on_data_received` 完整实现)
