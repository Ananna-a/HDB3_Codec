# 常见问题

## 通信类

### Q: 发送命令后没有应答 (AA 55 ...)
**可能原因**:
1. CDC 串口选择错误 → 检查 Windows 设备管理器 CDC 设备
2. CH340 串口选择错误 → 确认接收端口
3. 帧校验和不正确 → 检查 `serial_protocol.py:generate_command()` CS 计算
4. FPGA PLL 未锁定 → 检查 LED[0] 是否闪烁 (心跳)

### Q: 调试日志中出现 Bode 诊断 LED [5/6/7] 信号
这些 LED 是 V9.2 版本添加的 Bode 模块调试信号 (`src/debugger_top.v:1778-1790`)。当前 Bode 模块未实现，这些信号全为 0，LED 不会亮。

### Q: 为什么 CH340 上的数据有时乱序
`uart_tx_mux` 使用严格优先级仲裁，单字节粒度。如果高优先级通道 (Bode/频率/CAN_RX) 持续有数据，低优先级通道 (应答帧/蓝牙) 可能被饿死。这是设计特性，不是 bug。

## 示波器类

### Q: 示波器界面卡死
**诊断步骤**:
1. 降低采样率: `div_set=10` (5 MSPS)
2. 缩短 Buffer 长度: 10K 点
3. 检查 UDP 端口 6102 是否被占用
4. 检查以太网连接和 IP 地址

### Q: UDP 丢包严重
**优化方案**:
1. 编译 Cython 加速: `python setup_fast_udp.py build_ext --inplace`
2. 增大系统接收缓冲区: 已配置 8MB (`src/APP/core/ethernet_receiver.py:65`)
3. 降低采样率减少数据量
4. 检查网卡驱动和防火墙设置

### Q: 频率测量值异常
频率测量基于 1 秒硬件计数 (`frequency_counter.v`)，通过 CH340 发送 8 字节 (CH1+CH2, LE32)。上位机 `SerialManager` 需要 CD 收齐 8 字节才发射 `frequency_data_received`。如果中途收到其他数据包打断状态机，可能导致频率测量超时。重发 0x27 命令即可。

## 逻辑分析仪类

### Q: LA 采集后上位机收不到数据
**当前状态**: 逻辑分析仪采样引擎完成，但数据回传通道 (FPGA→PC) 尚未实现。这是正在进行的工作 (`src/logic_analyzer/` 模块)。

### Q: 看门狗超时 LED 指示
5 秒超时自动复位保护 (`src/debugger_top.v:1223-1238`)。如果 LA 状态机卡在非 IDLE 状态超过 5 秒，看门狗强制复位。正常使用不会触发。

## DDS 类

### Q: 频率不准
**检查**:
1. 上位机频率字计算: `freq_word = (f_hz << 32) / 125_000_000` — 确保使用 125MHz (不是 50MHz)
2. 精度: ±0.03 Hz (32 位累加器分辨率)
3. 如果使用 0x19/0x1A 批量设置 → 检查大端字节序

### Q: 任意波形上传后播放是正弦波
上传 256 字节数据后 (0x1E/0x1F)，需要**单独发送 0x10/0x11** 将波形类型切换为 6 (任意波形)。DDS_Param_Controller 不会自动切换。

## CAN 类

### Q: CAN 发送后没有应答
**检查**:
1. CAN 收发器 SIT1042 供电是否正常
2. CAN 总线终端电阻 (120Ω) 是否连接
3. 波特率: 当前固定 1 Mbps (`src/protocol/devices/can/can_controller.v:100`)
4. CAN UDP 通道已禁用 (RGMII 冲突) — 接收数据通过 CH340 UART 回传

### Q: CAN 接收数据看不到
SerialManager 自动识别首字节 0x00/0x01 为 CAN 帧并发射 `can_data_received` Signal。CAN 面板需订阅此信号才能显示。

## DDR3 类

### Q: DDR3 初始化失败 (LED[1] 不亮)
**可能原因**:
1. `ddr_pll` 未锁定 → 检查硬件时钟
2. DDR3 焊接/连接问题
3. IP 核参数不匹配 — 确认 GW5AT-138B 芯片型号

### Q: Stream 模式下偶尔丢点
DDR3 流控制有保护阈值: 接近满时停止写入 (`ALMOST_FULL_THRESHOLD = 254MB`)，接近空时停止读取 (`ALMOST_EMPTY_THRESHOLD = 32KB`)。如果流控触发，可能短暂丢点。通常不会影响正常显示。

## 开发类

### Q: 新增 V 文件后综合找不到模块
需在 `.gprj` 工程文件中注册新文件路径。使用 Gowin EDA 的文件管理器添加，不要手动编辑 `.gprj` XML。

### Q: 修改 IP 核后功能异常
Gowin IP Generator 重新生成后，`temp/` 子目录下的 `fifo_define.v` 和 `fifo_parameter.v` 会被覆盖。注意检查 FIFO 深度和位宽是否符合预期。
